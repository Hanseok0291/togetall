## 1. 배경 & 목표

### 1.1 배경

운동 파트너를 찾거나 러닝 크루를 모집하는 수요는 있으나, 정보가 카카오톡·인스타·커뮤니티 앱 등에 흩어져 있고 신뢰·안전 측면에서 불편이 있다.

### 1.2 제품 목표

- **함께 운동할 사람**과 **러닝 크루**를 찾는 경험을 한 앱으로 모은다.
- **운동 관련 커뮤니티 글**로 참여와 재방문을 유도한다.
- **MVP 1순위**는 [VISION.md](./VISION.md)에 정의된 대로 **파트너 탐색·연결**이다.

### 1.3 비목표 (이번 PRD 범위에서 명시하지 않는 것)

- 의료·부상 진단, 전문 코칭 과금 구조.
- 실시간 GPS 추적 공유(라이브 위치 공유).

## 2. 사용자 역할

| 역할 | 설명 |
|------|------|
| **일반 사용자** | 프로필을 등록하고, 파트너/모집 글을 검색·열람·작성하고 커뮤니티에 참여한다. |
| **크루장** (논리 역할) | 크루 모집 글을 올리고, MVP에서는 **글/댓글로 문의를 받는** 형태로 정의한다. Phase 2에서 가입 승인·크루 관리 기능을 확장할 수 있다. |
| **크루원** (선택) | MVP에서는 별도 역할 구분 없이 “일반 사용자”와 동일하게 취급할 수 있다. |

## 3. 기능 요구사항

### 3.1 계정 & 프로필

- 이메일/소셜 등 **로그인 방식**은 구현 단계에서 선택(스택 문서 참고).
- 프로필 필드(최소): 닉네임, 운동 종목(복수 선택 가능), 활동 수준(예: 입문/중급/상급), 활동 지역(시·구 단위 권장), 선호 시간대(요일·시간대 텍스트 또는 선택).
- 프로필 사진은 선택.

### 3.2 파트너 (MVP 핵심)

- **모집 글 작성**: 제목, 본문, 종목, 지역, 일정/시간대.
- **목록 & 검색·필터**: 지역, 종목, 키워드(선택).
- **상세**: 본문, 작성자 프로필로 이동, 댓글.
- **관심/연결**: 최소 **댓글** 또는 **관심(좋아요)** 중 하나 이상으로 “대화 시작”을 측정 가능하게 함.

### 3.3 크루 (MVP: 단순 형태)

- 크루 모집은 **게시 유형 또는 카테고리**로 구분(예: “파트너” / “크루 모집” / “자유”).
- 크루 전용 **가입 승인 워크플로**는 MVP에서 필수 아님. 대신 본문에 연락 방법·외부 링크를 허용할지 정책으로 관리(스팸 방지 규칙과 함께).

### 3.4 커뮤니티 (MVP: 단순 형태)

- **피드**: 최신순(기본). 카테고리 필터 가능하면 우선순위 낮음.
- **글쓰기·댓글**: 운동 관련 주제. 삭제는 본인 글/댓글 기준.
- **신고**: 글·댓글·프로필에 대해 사유 선택 후 제출.

### 3.5 신뢰·안전 (필수)

- **차단**: 특정 사용자의 글/프로필 노출 제한.
- **신고**: 운영 검토를 위한 저장 및 접수 ID(구현 시).
- **온보딩/FAQ**: 공공장소 만남, 신원 확인 권장 등 **안전 가이드** 문구 노출.
- **개인정보**: 위치는 **구 단위** 등 과도한 정밀도 기본 비노출. 연락처는 사용자가 본문에 직접 적는 방식을 허용할 경우, 스팸·사칭 대응 정책을 운영 규칙에 명시.

## 4. 비기능 요구사항

| 구분 | 요구 |
|------|------|
| 언어·로캘 | 한국어 UI, 한국 시간대 기준 날짜 표시 |
| 접근성 | 시스템 글자 크기 존중(가능 시), 버튼 터치 영역 확보 |
| 알림 | MVP: 앱 내 알림 또는 이메일 중 최소 1채널(구현 시 결정). 푸시는 Phase 2 가능 |
| 동시 플랫폼 | **한 코드베이스(Next.js)**로 모바일·데스크톱 브라우저 대응. 네이티브 앱은 별도 검토 |
| 보안 | 비밀번호·토큰은 클라이언트에 평문 저장 금지, HTTPS |

---

## 5. 하지 않을 것 (MVP 명시)

- 유료 멤버십·인앱 결제.
- 복잡한 추천 알고리즘(협업 필터 등).
- 실시간 위치 공유 맵.
- 크루 내부 전용 채팅방(Phase 2 후보).
