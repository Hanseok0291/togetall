# Togetall — 1페이지 비전 & MVP 범위

## 한 줄 정의

**Togetall**은 혼자가 아닌, 함께 운동할 사람과 러닝 크루를 찾고 운동 이야기를 나누는 **웹 기반 커뮤니티**(모바일 화면 우선) 서비스다. (브랜드명은 *Together*의 변형.)

## 해결하려는 문제

- 근처에서 **비슷한 강도·시간대**로 운동할 파트너를 찾기 어렵다.
- 러닝 크루 **크루장**이 크루원을 모집할 때 여러 앱에 흩어져 글을 올리게 된다.
- 운동 관련 **질문·후기·모집**이 한곳에 모이지 않아 신뢰와 지속 참여가 약하다.

## 핵심 가치

> 함께 운동할 사람과 크루를 **빠르게 찾고**, **안전하게 연결**하며, **운동 커뮤니티**로 남는다.

## MVP 1순위: **파트너 매칭(탐색·연결)**

첫 출시에서는 **“운동 파트너 찾기”**를 제품의 중심에 둔다.

- **포함**: 프로필(종목·수준·지역·시간대), 파트너 모집/검색·필터, 글 상세, 관심 표현(또는 댓글), 기본 신고·차단.
- **크루·커뮤니티**: MVP에서는 **단순한 형태**로만 포함한다.  
  - 크루: 모집 글을 “크루” 카테고리로 올리거나, 크루 페이지는 **읽기 위주 + 외부 연락/링크** 수준.  
  - 커뮤니티: **짧은 피드 + 글쓰기·댓글**까지는 가능하되, 추천 알고리즘·복잡한 게시판 구조는 제외.

이렇게 하면 개발·검증 주기를 짧게 가져가면서도 브랜드 스토리(함께 + 크루 + 커뮤니티)를 문서와 온보딩에서 일관되게 말할 수 있다.

## 성공의 첫 지표 (MVP)

- 주간 활성 사용자(WAU) 및 **파트너 관련 글/조회** 비율.
- **연결 이벤트**: 댓글 달기, 관심 보내기, 또는 프로필에서 연락 시도 등, 한 번이라도 “대화가 시작된” 세션 수.

## Phase 2 이후 (비전 문서 수준에서만 명시)

- 크루 전용 기능 강화(가입 요청·승인, 크루 채팅 등).
- 알림·푸시 고도화, 지역 기반 추천(개인정보 정책과 함께).
